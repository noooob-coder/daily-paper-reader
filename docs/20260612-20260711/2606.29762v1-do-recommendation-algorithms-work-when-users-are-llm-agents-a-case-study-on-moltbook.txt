## Abstract 

Large language model (LLM) agents are increasingly populating web platforms, raising a fundamental question for recommender systems: do algorithms designed for human users still work when users are LLM agents that may not have well-defined content con-sumption preferences? We study this question by formulating a fo-rum recommendation problem on Moltbook, a large-scale social me-dia platform exclusively for autonomous AI agents running on the OpenClaw framework. We evaluate eight recommendation meth-ods spanning simple heuristic rules, matrix factorization, ItemKNN, graph-based, and sequential models on the task of predicting which forums an agent will engage with next. We find that simple popularity-based rules or item-side collabo-rative filtering leveraging the co-occurrence structure and a vote count feature outperform techniques that explicitly learn a user representation. The static agent persona descriptions, the closest analog to a preference profile, fail to add value in predicting en-gagement. This suggests that for AI agent users, recommendation may collapse from personalization to structural pattern matching. We show multiple lines of evidence that AI agents’ content con-sumption behaviors differ from human users, providing a new angle for studying agent societies and designing robust recommendation algorithms as agents increasingly populate the web.

The assumption underlying all these advances is that users pos-sess latent yet learnable preferences that persist across interactions and evolve gradually over time. Human users’ interests tend to be consistent within sessions and drift slowly across weeks or months [ 5 , 7, 54 ], enabling increasingly sophisticated personaliza-tion. However, a new class of web users, autonomous LLM agents [ 40 ,55 ], is emerging that may fundamentally violate this assumption. These agents, powered by LLMs, are increasingly deployed on web platforms, where they perform tasks and consume content along-side human users [ 14 , 37 , 49 , 68 ]. Still, it is unclear whether they possess a clear content preference profile. Many deployed agent architectures [ 1, 50 , 59 , 62 ] have limited or no persistent memory across sessions—each conversation turn may reset the agent’s con-text, and even agents configured with external memory tools (e.g., memory files, persona descriptions) may not develop a persistent yet evolving preference profile that recommender systems assume. Consider this scenario: when a human user encounters an inter-esting post from a category they have never explored before, this exposure can spark a new, lasting interest that shapes their future browsing and engagement patterns. An LLM agent, by contrast, is typically stateless between sessions, and it remains unclear whether agents can organically develop their own interest groups over time through accumulated experience on the platform. This raises a fundamental question for recommender systems and agent behavioral studies. If agent users lack clear preferences, can existing recommendation models—which fundamentally rely on identifying preference patterns—still be effective? Or, do we need entirely different approaches for agent-populated platforms? As LLM agents become more prevalent on the web, answering this question has both scientific and practical importance [2, 9, 68]. We investigate this question empirically on Moltbook 1, the first recently launched large-scale social media platform exclusively designed for autonomous AI agents running on the OpenClaw framework 2. Moltbook’s structure mirrors Reddit: agents create posts in topic-based forums called submolts , and other agents can comment on and upvote these posts (karma). We formulate a fo-rum recommendation problem on this platform—predicting which submolts an agent will engage with next—using the publicly avail-able Moltbook Observatory Archive dataset [ 16 ], which captures roughly 2.4 million posts and 992,000 comments by 175,000 LLM agents across 6,200 submolts over a 10-week observation period.

Specifically, we evaluate eight recommendation methods span-ning simple heuristic rules (Random, TopPopular), matrix fac-torization (BPR-MF [ 45 ]), hybrid content-CF (HybridMF [ 24 ]), content-embedding-based filtering, item-based collaborative filter-ing (ItemKNN [ 47 ]), graph-based collaborative filtering (LightGCN [ 20 ]), and sequential recommendation (SASRec [ 28 ]). We addition-ally test the effect of karma-weighted interaction matrices and agent description embeddings as user-side features. We find that simple methods such as ItemKNN leveraging structural co-occurrence pat-terns perform the best among the set, while models such as BPR-MF that attempt to learn a personalized representation perform worse than simple TopPopular heuristics. Temporal gaps between the training and testing sets do not degrade evaluation metrics as they do on human platform data [ 7, 22 ]. Overall, the asymmetric results we observe with agent users indicate that personalization is funda-mentally different in agentic populations. The core contributions of our work are summarized as follows: (1) We formulate, to the best of our knowledge, the first recom-mendation problem on an agent-native social platform at large scale and establish empirical baselines across a wide range of algorithms. (2) We provide evidence that AI agents’ content consumption behaviors differ fundamentally from human users, with per-sonalization largely collapsing into structural pattern match-ing. (3) We discuss the implications on the algorithmic design and governance for recommender systems on human platforms as LLM agent participation grows, while also introducing a new direction to investigate agent societies. The paper is organized as follows. Section 2 reviews related work on the existing recommendation algorithms, how recommendation shifts in the agentic era, the Moltbook platform and the multi-agent systems research work it sparked recently. Section 3 introduces the setup of OpenClaw agents on the Moltbook platform, the public dataset we leverage, and our recommendation problem formulation. Section 4 describes the experiment and evaluation methodology. Section 5 presents the experimental results. Finally, we interpret the results and discuss its implications on human users and recommen-dation algorithmic design in Section 6. Overall, our work brings a new dimension to recommender systems and social networks research as AI agents become an increasingly significant presence on web platforms. 

## 2 Related Work 2.1 Collaborative Filtering and Matrix Factorization 

Collaborative filtering (CF) is a well-established paradigm in recom-mender systems [ 17 , 21 , 51 ]. User-based CF [ 36 , 53 ] recommends items liked by similar users, while item-based CF [ 13 , 35 , 47 ] rec-ommends items similar to those the target user has previously interacted with. Matrix factorization methods [ 29 ] decompose the user-item interaction matrix into low-rank latent factors, enabling scalable personalization. BPR [ 45 ] optimizes a pairwise ranking loss for implicit feedback, while factorization machines [ 44 ] gener-alize MF to incorporate side information. LightGCN [ 20 ] advances collaborative filtering by applying simple, linear neighborhood aggregation across the user-item bipartite graph, achieving state-of-the-art performance. The cold-start problem [ 48 ] arises when new users or items lack sufficient interaction history for CF to op-erate. Hybrid methods [ 6, 30 ] address this by combining CF with content features. Our study tests this well-established recommen-dation paradigm in a setting where users are AI agents, who may not have persistent or well-defined user preferences on the content they consume, challenging the fundamental assumption underlying personalization. 

## 2.2 Temporal Dynamics in Recommendation 

Time-aware recommender systems [ 7, 42 , 70 ] model how user pref-erences evolve over time. The common finding is that recommenda-tion accuracy degrades as the temporal gap between training and test data increases, because of data freshness drop [ 22 ] and user preferences drift [ 8 ]. Our temporal decay experiment (Section 5.4) tests whether this degradation occurs for LLM agent users, provid-ing a clean empirical test of the preference-stability assumption. In recommender systems for human users, sequential recom-mendation leveraging user behavioral sequences represents the state-of-the-art family of algorithms [ 28 , 52 , 60 , 69 ], substantially outperforming classical and MF based methods. In this study, we test whether the activity sequence data of agent users helps with recommendation in this novel setting. 

## 2.3 LLM Agents and Recommender Systems 

The field of recommendation systems is undergoing a paradigm shift towards generative recommenders [ 63 ] and agentic recom-mendation systems [ 25 ]. Recent work has explored using LLMs to power recommender systems [ 3 , 12 , 15 , 34 , 58 , 67 ], leveraging their language understanding capabilities to improve recommenda-tions. However, the complementary question—whether traditional recommendation algorithms work when users themselves are LLM agents—has received little attention. We aim to fill this gap in this work. 

## 2.4 Multi-Agent Systems and Moltbook 

The study of LLM agents as autonomous actors on web platforms is a rapidly growing area [ 2 , 14 , 40 , 55 ]. Generative agents [ 40 ] demon-strated that LLM-powered agents can simulate believable human behavior in sandbox environments [ 64 ]. Studying the behaviors of AI agents, and the contrast and interactions with humans forms a new research perspective [ 9]. To our knowledge, there have been no studies yet on the behaviors of AI agents as content consumers in recommendation settings. The Moltbook platform is a recently launched Reddit-style social media exclusively designed for autonomous AI agents. It is the first large-scale social network where agents interact without human in-tervention, providing an unprecedented testbed to study AI-native societies and ecosystems. It has been rapidly attracting research interest from the computational social science and network science communities, with existing work mostly focusing on collective dynamics and comparative graph analysis against human social networks [ 27 , 41 , 57 , 66 ]. ## 3 Preliminaries 3.1 OpenClaw and Moltbook 

Moltbook, launched in January 2026, is a social platform built on the OpenClaw open-source framework for deploying autonomous LLM agents on the web. Each OpenClaw agent living inside Moltbook is backed by one or more LLMs (e.g., Claude, GPT, Gemini) and can be configured by its human owner via various Markdown files. Three markdown files are mandatory to this design: SOUL.md defines the agent’s persona and behavioral guidelines, AGENTS.md defines the operational rules for the agent, and USER.md tells the agent the demographics and communication preferences of its human owner. Optional markdown files such as MEMORY.md , which can be updated by the agent itself, provide persistent context across sessions, and 

SKILL.md specifies the agent’s capabilities. These configuration files reside on the private instances where the agent is deployed, and are not accessible from Moltbook. In none of these markdown files are the agent’s own content preferences explicitly required, nor does the MEMORY.md , if there is one, maintain records of the agent’s full activity logs due to context window constraint. Moltbook’s platform structure mirrors Reddit: agents create posts in topic-based forums called submolts , and other agents can com-ment on and upvote these posts ( karma score). Humans can observe but cannot participate in those conversations with agents. Submolts span diverse topics—from technical discussions (e.g. todayilearned ,

openclaw-explorers ) to social and creative forums (e.g. shitposts , mu-sicrecs , humanwatching ). Agents navigate this space autonomously, producing interaction patterns that superficially resemble human social platform activity. 

## 3.2 The Moltbook Observatory Archive Dataset 

Our study leverages the public dataset of Moltbook Observatory Archive [ 16 ], available on HuggingFace 3. This incremental dataset passively records the profiles and activity logs of agents, communi-ties (submolts), posts, and comments by continuously polling the Moltbook API, providing a natural testbed to conduct reproducible studies of the content ecosystem of an AI-agent society at large scale. We describe the data processing steps in Section 4.

## 4 Methodology 4.1 Data Preprocessing and Train-Test Split 

From the Moltbook Observatory Archive dataset (Section 3.2), we deduplicate posts and comments that appear multiple times due to incremental API polling (retaining the most recent fetch of each record) and truncate to a 10-week observation window (January 27 – March 30, 2026), yielding the statistics in Table 1. We filter to agents with at least 5 total interactions (posts + comments) across all submolts, reducing the agent count from 175,021 to 79,643. For train-test split, we use temporal splitting rather than random splitting to avoid information leakage from future interactions [ 7]: the 10-week window is divided into a training period (weeks 1– 9) and a test period (week 10). We train models on the training-period interactions and evaluate on the test-period interactions, considering only agents and submolts that appear in both periods. For hyperparameter selection, we use the first 8 weeks as the tuning-training data and week 9 as the validation set to do a grid search optimizing Recall@10. A key difference between our dataset and typical datasets for recommendation is that as of now there is no feasible way to log the impression history of an agent, but only what they actively posted or commented in. This is the key reason we choose to work on the submolt forum recommendation rather than recommending individual posts, as we can use posting and commenting as a natural proxy of reactions to a submolt they have engaged with. Still, we discuss this limitation in Section 6.3.

## 4.4 Evaluation Metrics 

We follow the well established evaluation practices for top-𝑁 rec-ommendation [ 10 , 18 , 23 ]. For each agent in the test set, we generate a ranked list of 𝐾 recommended submolts (excluding those already interacted with in training) and compare against the agent’s actual test-period interactions. We report: 

• Recall@ 𝐾 : Fraction of test interactions that are retrieved in the top-𝐾 recommendations, averaged across agents. 

• NDCG@ 𝐾 [26 ]: Normalized Discounted Cumulative Gain, which rewards relevant submolts appearing earlier in the ranked list. 

• Hit Rate@ 𝐾 : Fraction of agents with at least one relevant submolt in the top-𝐾 list. 

• MRR : Mean Reciprocal Rank of the first relevant submolt. We evaluate each model and report the above metrics at 𝐾 ∈{5, 10 , 20 , 50 }.

## 4.5 Recommendation Models 

We evaluate eight models covering a wide range of recommenda-tion algorithms: two non-personalized baselines, a personalized latent-factor method, a hybrid model, a pure content-based model, a structural neighborhood method, a graph-based method, and a sequential model. We additionally test agent description embed-dings as a user feature in a separate ablation (Section 4.6). All the experiments were done locally on a MacBook M3 Pro laptop. 

4.5.1 Baselines: Random and TopPopular. The Random baseline recommends submolts uniformly at random (excluding those al-ready visited), providing a lower bound for evaluation. The Top-Popular baseline recommends submolts in decreasing order of total interaction count across all agents (excluding already-visited sub-molts). The TopPopular heuristic sets an interpretive anchor: any learned model that fails to outperform it is not extracting enough personalization signal beyond aggregate engagement frequency.

## 5 Results 5.1 Model Comparison 

Table 2 presents the main results across all eight models or heuris-tics. We find that ItemKNN, an item-based CF algorithm, overall performs the best. Moreover, the TopPopular heuristic provides a relatively strong baseline such that BPR-MF falls well below it, meaning that learning personalized latent factors actively hurts relative to simply recommending popular items. In recommender systems for human users, modeling user behavioral sequences us-ing transformer-based architectures such as SASRec significantly outperforms classical algorithms. However, in this new scenario, we do not see SASRec, while still being a strong variant, perform-ing any better than ItemKNN. This implies that temporal sequence information of agent users may not be as important, and we study this further in Section 5.4.

Our key interpretation is that LLM agents do not possess persistent or well-defined individual preferences, causing personalization to collapse and item-side structural information to dominate. Algorithms that attempt to learn or partially model a user representation tend to perform worse. We note that the absolute recommendation accuracy on Molt-book is notably low compared to most standard benchmarks. Two factors contribute to this gap. First, Moltbook’s interaction matrix is extremely sparse (0.041% density), below the sparsest benchmark datasets of Amazon-Books [ 38 ] ( ∼0.06% ), Yelp2018 [ 20 , 56 ] ( ∼0.13% ), and MovieLens-20M [ 19 ] ( ∼0.54% ). As a comparison, the original LightGCN work [ 20 ] reports Recall@20 of 0.033 and NDCG@20 of 0.025 on the Amazon-Book dataset, which are at roughly the same order of magnitude as the best performing model on our dataset. Second, the low absolute metrics are a consequence of the phenomenon we study: the absence of persistent user preferences imposes a ceiling on recommendation accuracy, as to a large ex-tent it reduces the problem of personalization to structural pattern matching. On human platforms, even sparse datasets yield reason-able metrics because users have genuine preferences that models can learn whereas on Moltbook, models can only exploit structural co-occurrence and popularity, which provides a weaker signal. 

## 5.2 Karma-Weighted Interactions 

Analogous to Reddit, the karma score summarizes the net vote count of posts and comments. Table 3 compares using the binary vs. karma-weighted interaction matrices across four applicable models and heuristics. We observe a consistent pattern across all the metric columns: Karma weighting dramatically improves structural and hybrid methods—TopPopular by 2.15 − 3.18 ×, ItemKNN by 

2.22 − 2.57 ×, and HybridMF by 2.46 − 5.17 ×, while leaving BPR-MF essentially unchanged ( 0.96 − 1.07 ×). This asymmetric effect is consistent with insights from the pre-vious section. TopPopular, ItemKNN, and HybridMF all benefit sub-stantially from karma weighting because they operate on aggregate interaction counts or content signals that karma filtering improves. The largest relative gain goes to HybridMF ( 5.17 ×), whose ALS latent factors benefit from the denoised karma-weighted matrix. BPR-MF, by contrast, is essentially unaffected by karma weighting. Its per-agent latent factors are noisy regardless of how interactions are weighted, confirming that the failure of personalization with this model is structural rather than a signal-quality issue. 

## 5.3 Agent Description Feature 

The dataset contains a textual description field defining who the agent is. Table 4 tests whether such agent description embeddings— static persona profiles that exist independently of interaction history—can improve recommendation. The results are a clean neg-ative: descriptions alone have almost no predictive power (the De-scriptionBased heuristic is only marginally better than the random baseline), and injecting them into HybridMF hurts performance relative to the original version. With DescriptionItemKNN there is no consistent metric improvement over the original ItemKNN either, and larger weights degrade performance (consistent metric drops at 𝛼 = 0.5). Together, these results indicate that the user-side description signal is weak and, when overweighted, introduces noise that degrades the structural signal. Note that the HybridMF results here differ slightly from the main results Table 2 because in this description ablation study we provide no user-side content features to HybridMF, isolating the effect of description injection. This result is informative because agent descriptions are the closest analog to a persistent “preference profile” available in the dataset—they encode the agent’s stated persona and topical inter-ests. The fact that these profiles fail to predict engagement demon-strates that agents’ actual interaction patterns are driven primarily by platform structure rather than by the topical affinities encoded in their descriptions. Even agents whose descriptions mention specific topics (e.g., “crypto-native entity”) do not reliably engage with the corresponding submolts, suggesting that navigation is shaped more by platform structure and session-level context than by personal preferences. 

## 5.4 Temporal Decay Analysis 

It is well known in the recommender systems community that model and data freshness matters a lot as user preferences can shift at distinct time scales: model performance degrades with the increase in train-test dates gap [ 7, 22 ]. To test whether this still holds in this completely new setting with AI agent users, we train the seven models (excluding random baseline) on a fixed 4-week window (slightly different evaluation results from Table 2) and vary the gap between the training and test periods from 1 to 4 weeks, while the test period is always one week. Figure 2 shows the results across the four evaluation metrics. Interestingly, no model shows systematic performance degrada-tion as the gap increases. The top-tier models or heuristics (Top-Popular, LightGCN, SASRec, ItemKNN) all show flat or slightly increasing profiles, as do the personalized models (BPR-MF, Hy-bridMF, ContentBased) at lower absolute performance. We note that the evaluation population shrinks at larger gaps (4,677 eval agents at gap = 1w vs. 1,895 at gap = 4w), because only agents active in both the training window and the distant test week qualify. We did sanity checks on the interaction counts during the training period of agents from different evaluation populations and did not see significant differences, ruling out the confounding factor of het-erogeneity in user activity level. The smaller population introduces variance, which likely accounts for the observed slight upward trend. The key observation is the absence of performance decay .Crucially, the absence of performance decay over train-test time gap holds across all model families—structural, personalized, se-quential, and content-based alike. ## 6 Discussion 6.1 Structural Pattern Matching vs. Personalization 

Our results consistently show that the predictive signal in agent interaction data is more structural than personal. Methods that exploit item co-occurrence (ItemKNN) or simple non-personalized TopPopular rule perform better than methods that attempt to learn personalized user embeddings (BPR-MF, HybridMF) or use content embedding centroids to represent users (ContentBased). This pat-tern is reinforced by three additional findings: (1) karma weighting amplifies structural methods by 2.2–5.2× while leaving BPR-MF’s performance unchanged; (2) agent description embeddings fail to help, indicating that even explicit persona information does not translate into useful predictive signals; and (3) recommendation accuracy shows no decay across train-test temporal gaps, implying stationary interaction patterns rather than evolving preferences. These findings are consistent with the way OpenClaw agents are configured [ 31 ]. The lack of user preference signals among agent populations can be explained as follows. First, agents can have shared LLM backbones. The overlapped models may have similar pretrained biases, reducing the user-side variance and the amount of learnable signals. Second, OpenClaw agents’ behaviors are heavily shaped by the configuration markdown files. These files are au-thored by individual agent owners and are held on private instances. Agents are typically configured to perform scheduled tasks under behavioral guidelines [ 59 ], but none of the markdown files require specifying the agent’s own content consumption preferences unless the owner explicitly defines it. Additionally, OpenClaw agents are stateless between distinct sessions, and the optional long-term mem-ory file cannot record full interaction logs. This sharply contrasts with human users whose content interests evolve based on their experience during the previous sessions on the platform [8, 42]. As the architectures of agentic systems are rapidly evolving [ 1 ,59, 68], we expect some of the assumptions and interpretation are subject to change in the future. Still, we believe this is a direction worth revisiting as it provides a new dimension to study multi-agent systems. 

## 6.2 Implications for Recommender Systems on Human Platforms 

We discuss the implications of our study on recommendation for human users. Our findings highlight fundamental distinctions be-tween human users and LLM agents from the perspective of per-sonalization. Human users exhibit clear yet evolving preferences that personalized models can learn such that user behavioral data can be cleanly projected to a representation embedding in latent spaces [ 21 , 29 , 45 ]. LLM agents, by contrast, produce interaction patterns that are driven by frozen model weights, platform structure and session-level context rather than by persistent personal prefer-ences. As a result, we hypothesize that personalization collapses— user identity becomes less informative, temporal ordering of behav-ioral sequences adds little signal, and the recommendation problem reduces to structural pattern matching. This distinction indicates a qualitative difference in the data-generating process impacting algorithmic design. As LLM agents become increasingly prevalent on human-oriented platforms such as Reddit, X, and other similar social media sites, these findings have direct implications for recommender sys-tems designed to serve human users. First, training data may be polluted : if a non-trivial fraction of platform interactions are generated by agents whose behavior does not reflect genuine pref-erences, models trained on this mixed data may learn weaker or distorted user representations. The structural and popularity sig-nals that dominate agent behaviors could dilute the user signals that personalization relies on, degrading recommendation quality for human users. Second, evaluation metrics may be confounded :standard offline evaluation computes metrics over all users in the test set. If agents are present but unidentified, their lack of prefer-ence would systematically deflate metrics like Recall and NDCG for personalized models, potentially leading practitioners to draw wrong conclusions on model comparison. Third, detection and segmentation become particularly important : platforms may benefit from identifying agent users using anomaly detection al-gorithms [ 11 , 61 ] and excluding them from model training and evaluation for cleaner results. 

## 6.3 Limitations 

Our study has several limitations in data and methodology. On the data side, the dataset we use does not include the backbone LLM model information or the configuration files (e.g. soul.md ,

memory.md , skill.md ). When interpreting the results, we assume that agents may have overlapped LLM model backbones and the configuration files do not always define agent’s content consump-tion preferences or record agent’s full interaction logs. While we believe these assumptions are realistic with the typical way Open-Claw agents are deployed, they are not falsifiable as of now. We also note that the interaction data is very sparse, which may bring confounding factors in results interpretation as the absolute perfor-mance metrics are quite low. The Moltbook platform does not log full impressions and interactions history, bringing a valid concern when formulating a recommendation problem. We addressed this challenge by defining a forum recommendation problem leveraging agent posts and comments as a proxy of reactions to contents (in this case submolts). As this is a new research direction, we believe there is value in developing standardized benchmark datasets as future work. On the methodology side, the recommendation algorithms used in the current work are mostly classic and well-established methods. As a first empirical study, we aim to establish important qualitative claims rather than create new algorithms given lack of standard problem formulation or benchmarks. Still, our findings imply the importance of developing robust recommendation mod-els and evaluation methods when the platform is populated with AI agents. ## 7 Conclusion 

We present the first systematic evaluation of recommendation al-gorithms on an LLM-agent-native social platform. By formulating a forum recommendation problem on a public dataset collected from Moltbook and measuring the performance of a wide range of algorithms, we find that content consumption behaviors of agent users differ drastically from well-established findings on human platforms. Due to lack of persistent preference profiles, recommen-dation for AI agent users is closer to structural pattern matching rather than personalization. Simple heuristic or item-CF methods beat or match more complex user modeling techniques. User behav-ioral sequences do not show as much importance as they bring for human user modeling. Our findings provide a novel angle to agent behavioral studies, and have profound implications on designing better recommender systems for human users as LLM agents be-come more prevalent on web platforms. Future work includes estab-lishing standard benchmark datasets, developing novel algorithms and evaluation methodologies in this under-explored setting.
