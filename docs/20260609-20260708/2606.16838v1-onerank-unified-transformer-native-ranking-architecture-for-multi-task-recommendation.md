## 摘要
多任务学习在推荐系统中至关重要，能够实现不同用户反馈之间的互补学习。尽管现代工业实践已从DNN转向以Transformer为中心的架构以增强序列建模和扩展能力，但它们仍然将特征编码与多任务预测解耦，将Transformer视为任务无关的编码器。这种设计从根本上限制了性能和可扩展性，具体表现为：(1) 在异构任务目标下形成信息瓶颈，(2) 引发梯度干扰导致跷跷板现象，(3) 迫使数据流转换，使基于注意力的上下文自适应表示学习转化为具有不兼容信息读写动态的静态前馈任务预测。我们提出OneRank，一种原生Transformer多任务排序框架，消除了编码器-预测器分离，并为前向表示学习和反向优化引入任务私有通道，实现任务专门化学习并减少任务间干扰。在前向过程中，OneRank通过任务条件信息选择、候选项感知上下文化和受控的跨任务交互，自下而上地学习任务特定表示。在反向过程中，跨任务梯度分离将任务私有参数更新与共享知识提取模块隔离，防止负迁移。我们进一步用基于动态匹配的打分替代静态任务特定MLP评分器，实现上下文感知的个性化排序。通过将多任务推理内化到Transformer栈中，OneRank建立了一种统一且可扩展的架构范式。在大规模工业数据集上的离线和在线实验表明，OneRank在保持计算效率的同时显著优于现有基线方法。

## Abstract
Multi-task learning (MTL) is essential in recommender systems to enable complementary learning among diverse user feedback. While modern industrial practices have shifted from DNNs to Transformer-centric architectures to strengthen sequence modeling and scaling capacity, they still decouple feature encoding from multi-task prediction, treating the Transformer as a task-agnostic encoder. This design fundamentally limits the performance and scalability by (1) creating an information bottleneck under heterogeneous task objectives, (2) inducing gradient interference that leads to the seesaw phenomenon, and (3) forcing a dataflow transition in which attention-based, context-adaptive representation learning is converted to static feed-forward task prediction with incompatible information read-write dynamics.   We propose OneRank, a Transformer-native multi-task ranking framework that eliminates encoder-predictor separation and introduces task-private channels for forward representation learning and backward optimization, enabling task-specialized learning while reducing inter-task interference. In the forward pass, OneRank learns task-specific representations bottom-up through task-conditioned information selection, candidate-aware contextualization, and controlled cross-task interaction. In the backward pass, cross-task gradient detachment isolates task-private parameter updates from shared knowledge extraction modules, preventing negative transfer. We further replace static task-specific MLP scorers with dynamic matching-based scoring for context-aware personalized ranking. By internalizing multi-task reasoning within the Transformer stack, OneRank establishes a unified and scalable architectural paradigm. Offline and online experiments on large-scale industrial datasets show that OneRank significantly outperforms state-of-the-art baselines while maintaining computational efficiency.

## 1. 核心问题与整体含义（研究动机与背景）

- **背景**：多任务学习是现代推荐系统的核心范式，通过联合建模不同用户反馈（如点击、加购、下单）实现互补学习。近年来，工业界从 DNN 转向 Transformer 架构以增强序列建模和扩展能力。
- **问题**：现有 Transformer 多任务推荐架构仍保留 **编码器‑预测器分离**（encoder‑predictor decoupling）的设计，即 Transformer 只作为任务无关的编码器，下游用静态 MLP 塔做任务预测。该设计存在三个根本缺陷：
  - **信息瓶颈**：共享表示无法区分任务特定信号与共享知识，迫使下游解码器从混合嵌入中解离任务信息。
  - **梯度干扰（跷跷板现象）**：共享底层的梯度冲突导致部分任务性能相互损害。
  - **数据流范式不匹配**：Transformer 的上下文自适应注意力编码与 MLP 的静态非线形预测不兼容，阻碍端到端可扩展推理。
- **论文含义**：提出 **OneRank**，一种原生 Transformer 多任务排序框架，将多任务推理内化到 Transformer 堆栈内部，消除编码器‑预测器分离，实现统一建模与可扩展计算。

## 2. 方法论

### 2.1 核心思想
- **前向过程**：自底向上构建任务私有通道与任务共享通道。
  - **输入层**：引入任务特定 token（task‑specific tokens）并设置相互不可见（mutual invisibility），实现早期任务专化。
  - **中间层**：通过**候选项感知上下文化**（candidate‑aware contextualization）聚合跨候选项信号，使用场景描述符（Situational Descriptor）桥接训练‑服务鸿沟。
  - **预测层**：通过**受控跨任务关系注意力**（cross‑task relational attention）选择性注入任务依赖，并配合**梯度分离**（gradient detachment）实现前向知识传递、后向优化隔离。
- **反向过程**：跨任务注意力中的梯度分离，将任务私有参数更新与共享知识提取模块隔离，防止负迁移。
- **动态匹配打分**：用**内积相似度**替代静态 MLP 打分，使全局任务表示与上下文条件的候选项嵌入动态匹配，实现上下文感知个性化排序。

### 2.2 关键技术细节

#### 2.2.1 结构化标记化（Structured Tokenization）
- 输入被组织成统一 token 序列，包括：
  - **交互历史（IH）**：用户行为序列 + 位置编码。
  - **偏好锚定（PA）**：从外部知识检索的序列（如历史高点击/高购买序列），用 `<BOS>`/`<EOS>` 包裹。
  - **候选项‑任务 Token 组**：每个候选项嵌入 + K 个共享任务 token（作为查询模板）。
- 总序列长度：`S = T + Σ|A_i| + 2M + N·(1+K)`。

- **结构化注意力掩码**：约束任务 token 只能关注用户上下文、自身及同组的候选项嵌入，不同组之间隔离，不同任务 token 之间相互不可见。公式 (4) 给出掩码定义。
- **Transformer 编码**：L 层掩码多头自注意力 + 残差 + 层归一化。
- 从每个候选项组中提取各任务 token 的输出作为任务特定表示 `r_i^k`。

#### 2.2.3 候选项感知上下文化（Candidate‑Aware Contextualization）
- 使用场景描述符 s（含用户画像、查询、会话元数据）作为查询锚点。
- 每个任务 k 有自己的投影函数 `f_k(·)` 将 s 映射到查询 `q_k`。
- 通过任务特定多头交叉注意力（MHCA_k）聚合所有候选项的任务表示 `{r_i^k}` 得到全局任务表示 `h_k`。

#### 2.2.4 多任务预测（Multi‑Task Prediction）
- **跨任务注意力**：对全局任务表示 `{h_k}` 应用可配置掩码 A 的自注意力。
- **策略梯度分离**：在后向传播中仅允许对角线梯度流动，阻止非对角梯度，实现前向知识传递、后向优化隔离。
- **动态匹配打分**：`s_i^k = z_k^T · r_i^k`，其中 `z_k` 是经过 FFN 和残差后的任务全局表示；`r_i^k` 是任务特定候选项表示。
- **跨任务掩码策略**：可灵活配置为并行掩码（独立）、空掩码（全连接）、级联掩码（如 click→cart→purchase）、混合掩码。

### 2.3 整体流程文字描述
1. 输入被转换为统一 token 序列，包括用户上下文和多个候选项‑任务组。
2. 经过 L 层带结构化掩码的 Transformer 编码，每个任务 token 从对应候选项和共享上下文中提取任务特定表示 `r_i^k`。
