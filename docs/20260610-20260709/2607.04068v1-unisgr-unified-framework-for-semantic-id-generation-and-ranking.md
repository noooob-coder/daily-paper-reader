## 摘要
推荐系统在现代电商平台中扮演关键角色。生成式检索作为一种新兴范式，有助于缓解多阶段级联架构的局限性，但现有方法在处理细粒度多目标排序时仍存在不足。为此，我们提出UniSGR，一种用于语义ID生成与排序的统一框架。UniSGR采用两阶段训练范式：先进行多场景预训练阶段，从混合业务场景数据中学习，然后进行场景特定对齐阶段，联合优化价值感知并行多令牌预测（VA-PMTP）和统一多目标排序模块。为了更好地将生成与下游排序对齐，我们引入了由漏斗感知对比学习引导的任务感知令牌（TAT）。此外，我们提出带有重组织KV缓存的语义树注意力（STARK），这是一种消除传统束搜索中关键效率瓶颈的推理策略。在大型电商平台上进行的大量离线实验证明了UniSGR的有效性和可扩展性。

## Abstract
Recommendation systems play a pivotal role in modern e-commerce platforms. While generative retrieval has emerged as a promising paradigm for alleviating the limitations of multi-stage cascade architectures, existing methods still struggle with fine-grained multi-objective ranking. To bridge this gap, we propose UniSGR, a Unified framework for Semantic ID Generation and Ranking. UniSGR adopts a two-stage training paradigm: a multi-scenario pre-training stage that learns from mixed business-scenario data, followed by a scenario-specific alignment stage that jointly optimizes Value-Aware Parallel Multi-Token Prediction (VA-PMTP) and a unified multi-objective ranking module. To better align generation with downstream ranking, we introduce Task-Aware Tokens (TAT) guided by Funnel-Aware Contrastive Learning. Furthermore, we propose Semantic Tree Attention with Reorganized KV cache (STARK), an inference strategy that removes key efficiency bottlenecks in conventional beam search. Extensive offline experiments on a large-scale e-commerce platform demonstrate the effectiveness and scalability of UniSGR.

## 1. 核心问题与研究动机

- **背景**：现代电商推荐系统通常采用多阶段级联架构（检索→预排序→排序→重排序），这种分阶段优化导致各模块只关注局部子任务，而非端到端推荐目标。下游排序器只能对上游检索过滤后的候选集进行重排序，早期被过滤掉的相关物品无法被恢复，限制了真正端到端的推荐能力。
- **现有不足**：生成式检索（如TIGER、OneRec等）通过自回归生成语义ID来直接生成候选物品，提供了更统一的候选生成视角。但现有方法在细粒度多目标排序（如点击、加购、购买）上能力有限，生成器与下游排序器之间存在目标不一致（objective mismatch）。
- **核心问题**：如何在一个统一框架内同时实现高质量的语义ID生成与多目标排序，消除级联架构的固有缺陷，并且支持多场景、多目标的联合优化。

## 2. 提出的方法论

### 2.1 整体思想
UniSGR是一个统一的生成-排序框架，采用**两阶段训练范式**：
- **多场景预训练**：在混合多个业务场景的数据上进行下一物品预测（Next Token Prediction, NTP），让模型学习跨场景的通用用户兴趣和物品语义。
- **场景特定对齐**：在目标场景上联合优化**价值感知并行多令牌预测**（VA-PMTP）和**统一多目标排序模块**，使生成结果与业务目标对齐。

### 2.2 关键技术细节
- **分词器**：先利用多模态大模型（Qwen3-VL）和对比学习得到协作感知的物品嵌入，再通过RQ-VAE和Sinkhorn-Knopp均衡分配得到3层语义码本（每层大小为8192）。
- **模型架构**：非对称的Transformer编码器-解码器结构。
  - **编码器（MemoryNet）**：轻量级线性复杂度，将用户特征处理为前缀令牌，历史物品用语义ID嵌入加行为/位置信息表示，输出静态记忆表示。
  - **稀疏MoE解码器**：自回归生成语义ID令牌，使用分组查询注意力（GQA）减少KV缓存，采用共享SwiGLU-MoE层（路由专家+共享专家）提高容量和效率。
  - **令牌类型嵌入**：编码语义ID的层级结构。
- **两阶段训练中的关键组件**：
  - **VA-PMTP**：在同一个session内并行处理多个行为目标（点击、加购、购买）的语义ID序列，对不同行为赋予不同业务价值权重（购买>加购>点击），采用自定义并行掩码（同物品内因果，跨物品互不可见，共享BOS令牌），生成偏向高价值行为的候选集。
  - **统一多目标排序模块**：复用生成过程的编码器/解码器表示、语义ID表示，通过目标注意力（Target Attention）和渐进式分层提取（PLE）模块实现多目标BCE分类。排序损失与生成损失联合优化。
  - **任务感知令牌（TAT）**：在解码器输入中预置三个可学习的任务令牌（对应点击、加购、购买），使语义ID表示能感知目标偏向。再通过**漏斗感知对比学习**（FACL）对每个任务令牌进行对比学习：正样本分别为对应行为的物品，负样本按漏斗难度递增（点击：随机物品；加购：点击但未加购；购买：点击但未购买），防止语义漂移。
  - **联合损失**：`L = L_gen + α*L_rank + β*L_aux`，其中L_gen为VA-PMTP交叉熵，L_rank为多目标BCE加权和，L_aux为FACL对比损失。
- **推理加速（STARK）**：束搜索时，传统方法沿批次维度复制KV缓存导致大量冗余。STARK将候选路径沿序列维度拼接，使用预定义的树注意力掩码（仅允许令牌关注其祖先节点），共享前缀的KV缓存只存储一次，通过索引映射复用。该策略消除了数据复制和冗余前缀计算，吞吐量提升约200%。

### 2.3 公式/算法流程（文字说明）
- 预训练：最大化给定用户特征和历史序列条件下下一物品语义ID序列的生成概率。
- 对齐阶段：对同一个session内的多个行为目标的语义ID序列同时进行生成，生成损失按行为价值加权；同时，利用生成过程中的隐状态预测每个候选物品的点击、加购、购买概率，使用BCE损失；再加上任务令牌的对比学习损失。
- 推理：解码器通过束搜索生成候选语义ID，复用隐状态和语义ID表示，直接通过统一排序模块计算多目标得分，无需另设排序模型。
