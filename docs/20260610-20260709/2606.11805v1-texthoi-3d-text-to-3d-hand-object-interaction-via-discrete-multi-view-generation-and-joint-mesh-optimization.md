## 摘要
文本条件的三维生成在图像和孤立物体方面进展迅速，但生成手物网格仍然具有挑战性：输出必须保持语言语义、跨视图一致性、物体几何、铰接手部形状以及物理上合理的接触。我们提出TextHOI-3D，一个分阶段框架，使用生成的多视图观测作为文本条件视觉生成与几何感知手物恢复之间的显式接口。TextHOI-3D学习一个紧凑的VQ令牌空间用于固定相机的手物观测，通过CLIP条件视觉自回归模型从文本预测多视图视觉令牌，并通过先验初始化、多视图联合优化和防穿透细化恢复统一的手物网格。该设计将语义生成与几何恢复分离，同时通过离散多视图表示保持两个阶段的连接。在HO3D衍生的评估中，与单视图对应方法相比，多视图设置将物体倒角距离从17.26毫米降至4.92毫米，穿透体积从5.3721立方厘米降至0.2193立方厘米，同时改进了手部误差和表面F分数。这些结果支持多视图视觉令牌作为文本驱动的3D手物网格创建的有效中间表示。

## Abstract
Text-conditioned 3D generation has progressed rapidly for images and isolated objects, but producing a hand-object mesh remains challenging: the output must preserve language semantics, cross-view consistency, object geometry, articulated hand shape, and physically plausible contact. We present TextHOI-3D, a staged framework that uses generated multi-view observations as an explicit interface between text-conditioned visual generation and geometry-aware hand-object recovery. TextHOI-3D learns a compact VQ token space for fixed-camera hand-object observations, predicts multi-view visual tokens from text with a CLIP-conditioned visual autoregressive model, and recovers a unified hand-object mesh through prior initialization, multi-view joint optimization, and anti-penetration refinement. The design separates semantic generation from geometric recovery while keeping both stages connected by a discrete multi-view representation. On HO3D-derived evaluations, the multi-view setting reduces object CD from 17.26 mm to 4.92 mm and penetration volume from 5.3721 cm^3 to 0.2193 cm^3 compared with a single-view counterpart, while improving hand errors and surface F-scores. These results support multi-view visual tokens as an effective intermediate representation for text-driven 3D hand-object mesh creation.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：文本到3D手物交互（Hand-Object Interaction, HOI）生成面临多重耦合约束——必须同时满足语言语义一致性、多视图连贯性、物体几何完整性、铰接式手部形状正确性以及物理合理的接触（无穿透或漂浮）。现有文本到3D方法（如DreamFusion、MVDream）主要面向孤立物体或单张图像，难以处理手与物体之间的遮挡和接触细节。
- **研究动机**：直接端到端文本→网格非常困难，因为手物接触区域常被遮挡，且配对文本-网格交互数据稀缺。作者提出一种分阶段替代方案：先生成连贯的多视图视觉证据（多视图图像），再恢复统一手物网格。多视图图像比网格容易生成，却能提供轮廓、遮挡和接触线索。关键在于如何使多视图表示同时具备紧凑性、可控性和几何可用性。
- **整体含义**：TextHOI-3D将语义生成与几何恢复解耦，通过离散多视图token作为中间接口，证明了多视图条件在提高手物交互3D重建质量上的有效性。

## 2. 方法论
- **核心思想**：三段式框架：①学习固定相机多视图观测的离散VQ token空间；②用CLIP条件的视觉自回归模型从文本预测多视图token；③通过先验初始化、多视图联合优化和防穿透细化恢复统一手物网格。
- **关键技术细节**：
  - **离散多视图表示**：将N=6个固定相机RGB视图沿通道堆叠成H×W×18张量，用VQ-VAE编码为16×16的token图，码本大小4096，每个码字维度32。训练损失为重构损失+码本承诺损失。该表示用一个场景级编码器学习跨视图统计相关性，一个token对应一个多视图场景单元。
  - **文本条件多视图生成**：采用VAR（Visual Autoregressive）模型的next-scale预测范式，从粗到细生成多尺度token金字塔。CLIP文本编码器提取全局特征和序列特征，全局特征通过AdaLN（自适应层归一化）调制视觉隐藏状态，序列特征通过cross-attention注入。训练使用teacher forcing，推理时使用classifier-free guidance和top-k/top-p采样。
  - **网格恢复**：首先生成视图通过HSV分割提取手/物掩码，用Stable Diffusion LoRA进行双向修复（遮挡区域）。物体分支使用InstantMesh初始化，手部分支使用OmniHands先验和MANO参数模型。优化变量包括手部参数、物体位姿和轻量物体修正。联合优化目标包含多视图重投影损失、掩码一致性损失、接触损失（鼓励接近但非穿透）和穿透损失（惩罚负距离），分三阶段：全局对齐、交互修正、局部细化+防穿透后处理。

## 3. 实验设计
- **数据集与场景**：基于HO3D数据集（标准手物交互基准，含YCB-Video物体和配对几何）。通过PyTorch3D渲染固定相机多视图（6视角，Zero123++轨道风格：方位角步长60°，俯仰角20°/-10°，分辨率256×256）。共16291帧，训练/验证分割为14662/1629帧。
- **评估协议**：表示质量用PSNR/SSIM/LPIPS；物体几何用Chamfer Distance（CD）和F-score@5/10mm（经ICP对齐后，10k点采样）；手部用MPJPE/MPVPE（Procrustes对齐）；交互质量用穿透体积（PV，Monte Carlo 100k点采样）、穿透手部顶点百分比、平均/最大穿透深度、接触比率（顶点距离物体表面<τ）。
- **对比方法**：主要对比是单视图输入 vs 多视图输入（相同数据、权重、恢复模块和评估代码），单视图时物体用Zero123扩展为稀疏视图再InstantMesh，手部用单视图先验。没有与其他文本到3D HOI方法直接对比（因为Text2HOI等侧重运动生成且输入条件不同）。

## 6. 主要结论与发现
- 多视图观测显著提升物体几何：CD从17.26mm降至4.92mm，F@5从46.3%升至92.7%。
- 手部误差降低：MPJPE从1.46mm降至0.65mm，MPVPE从1.54mm降至0.70mm。
- 穿透体积大幅减少：从5.3721cm³降至0.2193cm³，同时保持非零接触比率（CR@2=0.37%, CR@5=1.44%），表明优化并非简单分离手物，而是抑制严重穿透并保留局部接触。
- 离散多视图token作为中间表示有效，实现了语义生成与几何恢复的解耦，且能定位失败来源（如生成视图不一致主要影响初始化，分割/修复主要影响恢复）。

## 7. 优点
- **方法设计模块化**：三段式分离生成与恢复，方便调试和替换组件（如未来可用更优的物体/手部先验）。
- **离散表示紧凑高效**：堆叠多视图作为单一样本处理，内存和时间接近单视图（3.33 GB / 150.1 ms/step vs 单视图3.32/148.5），但吞吐量达40 views/s（单视图仅6.7 views/s），避免顺序编码或空间拼接的高开销。
- **文本控制精细**：CLIP条件结合全局AdaLN和局部cross-attention，既能控制物体类别和交互类型，又能细化局部属性（如手柄、抓握提示）。
- **恢复流程完备**：结合分割、修复、参数化手部模型、多视图优化和防穿透后处理，输出物理合理的交互网格。

## 8. 不足与局限
- **评估规模有限**：恢复量化仅9个示例，缺乏统计显著性分析；未在HO3D完整测试集或真实世界数据上评估。
- **缺乏外部对比**：未与现有文本到3D手物方法（如Text2HOI、DreamFusion等）在同任务上公平比较，仅为内部单/多视图消融。系统级比较（如用户研究）也未进行。
- **依赖合成渲染数据**：VQ-VAE和生成模型仅在固定渲染参数下训练，泛化到真实拍摄的多视图或不同光照/背景可能下降。
- **未见复杂交互分析**：实验物体种类有限（YCB物体），未测试非刚性物体、多物体交互或精细手部动态。
- **潜在偏差风险**：生成视图可能受训练数据分布限制（如抓握姿态偏好特定方向）；网格恢复中HSV分割假设颜色稳定，真实场景可能失效。
