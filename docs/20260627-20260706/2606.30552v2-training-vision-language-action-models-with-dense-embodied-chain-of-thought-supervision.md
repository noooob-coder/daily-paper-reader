## 摘要
跨本体迁移在视觉-语言-动作（VLA）模型中仍然具有挑战性，因为不同机器人平台的低级状态和动作空间存在根本差异。我们观察到，操纵背后的高级认知过程，包括场景感知、物体识别、任务规划和子任务分解，在不同本体之间很大程度上是共享的。基于这一观察，我们提出了ZR-0，一个拥有26亿参数的端到端VLA模型，它使用密集的具身思维链（ECoT）监督来对齐视觉-语言模型（VLM）中的跨本体表示。ZR-0采用双流架构：预训练的VLM（系统2）在训练期间生成结构化的ECoT推理，而基于扩散Transformer的动作专家（系统1）通过流匹配产生连续的动作块。这两个组件通过交叉注意力耦合，并带有一个注意掩码，该掩码限制动作专家仅使用输入提示特征，从而在推理时可以完全跳过ECoT生成而不损失任何性能。ZR-0在ProcCorpus-60M上进行了预训练，这是一个大规模数据集，包含来自超过40万条轨迹的约6000万帧（约1000小时），密集的ECoT注释覆盖了96.8%的帧。我们在三个模拟基准上评估了ZR-0，涵盖单臂（LIBERO）、双臂（RoboTwin 2.0）和人形（RoboCasa GR-1 Tabletop）本体，以及在xArm平台上的真实世界实验，展示了在所有设置中的强性能。代码和模型检查点可在https://github.com/RUCKBReasoning/ZR-0获取。

## Abstract
Cross-embodiment transfer in vision-language-action (VLA) models remains challenging because low-level state and action spaces differ fundamentally across robot platforms. We observe that the high-level cognitive process underlying manipulation, including scene perception, object identification, task planning, and sub-task decomposition, is largely shared across embodiments. Based on this observation, we present ZR-0, a 2.6 billion parameter end-to-end VLA model that uses dense Embodied Chain-of-Thought (ECoT) supervision to align cross-embodiment representations within the vision-language model (VLM). ZR-0 adopts a dual-stream architecture: a pre-trained VLM (System 2) generates structured ECoT reasoning during training, while a Diffusion Transformer-based action expert (System 1) produces continuous action chunks via flow matching. The two components are coupled through cross-attention, with an attention mask that restricts the action expert to input prompt features only, enabling ECoT generation to be entirely skipped at inference without any performance loss. ZR-0 is pre-trained on ProcCorpus-60M, a large-scale dataset comprising approximately 60 million frames (approximately 1,000 hours) from over 400K trajectories, with dense ECoT annotations covering 96.8% of all frames. We evaluate ZR-0 on three simulation benchmarks spanning single-arm (LIBERO), bimanual (RoboTwin 2.0), and humanoid (RoboCasa GR-1 Tabletop) embodiments, as well as real-world experiments on the xArm platform, demonstrating strong performance across all settings. Code and model checkpoints are available at https://github.com/RUCKBReasoning/ZR-0.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：跨本体迁移在VLA模型中极具挑战，因为不同机器人平台的低级状态和动作空间（如自由度、控制接口）存在根本差异，导致格式级解决方案（如零填充、归一化）无法实现真正的语义对齐。
- **观察**：操纵背后的高级认知过程（场景感知、物体识别、任务规划、子任务分解）在不同本体之间很大程度上是共享的，这部分知识应是可迁移的。
- **整体含义**：提出利用密集的具身思维链（ECoT）监督作为对齐跨本体表示的信号，使VLM学习本体无关的推理，从而提升跨本体泛化能力。

## 2. 方法论
### 核心思想
- 采用**双流架构**，灵感来自认知框架中的系统1/系统2：
  - **系统2**：预训练VLM（初始化自Qwen3-VL-2B-Instruct），处理视觉观察和任务指令，生成结构化的ECoT推理序列（包含场景描述、进度评估、未来计划、原子子任务、目标物体边界框、离散动作token）。
  - **系统1**：基于扩散Transformer（DiT）的动作专家，通过流匹配（Flow Matching）生成连续动作块。
- **关键设计**：通过交叉注意力耦合两系统，并应用**注意掩码**，限制动作专家仅关注VLM输入提示特征（即任务指令和图像），**不关注ECoT token**。这使得推理时可以完全跳过ECoT的自回归生成，仅需单次VLM前向传播即可获得动作专家所需特征，不损失性能。

### 训练目标
- **ECoT生成**：标准下一个token预测损失 \( \mathcal{L}_{\text{ntp}} \)。
- **连续动作预测**：流匹配损失 \( \mathcal{L}_{\text{fm}} \)，对噪声动作块预测去噪向量场。
- 总损失：\( \mathcal{L} = \mathcal{L}_{\text{ntp}} + \alpha \mathcal{L}_{\text{fm}} \)，其中 \( \alpha \) 为平衡系数（预训练时α=5，后训练时α=1）。

### 推理流程
- 接收任务指令、图像观察和机器人状态，从高斯噪声初始化动作块，通过N步前向欧拉积分迭代去噪，得到预测动作块。无需生成ECoT文本。

### 其他技术细节
- 状态和动作维度统一填充至64维，并归一化；使用AdamW优化器，cosine学习率调度（峰值3e-5，终值3e-6），bfloat16混合精度训练，梯度裁剪1.0。

## 6. 主要结论与发现
- **ZR-0在三个仿真基准上均达到竞争性或最优性能**：LIBERO 97.8%（优于π0.5的96.9%），RoboCasa GR-1 Tabletop 69.3%（优于JoyAI-RA的63.2%），RoboTwin 2.0 Clean 88.70%/Randomized 87.98%（与LingBot-VLA相当，但预训练数据量小一个量级）。
- **ECoT预训练显著提升跨本体表示对齐**：消融显示移除ECoT预训练后LIBERO平均成功率下降2.1个点。
- **推理时跳过ECoT不损失性能**：得益于交叉注意力掩码设计，推理延迟低，适合实时控制。
- **ECoT对常见技能（如拾放）提升明显，但低频技能（如关柜门）提升有限**：反映预训练数据分布的影响。

## 7. 优点
- **创新性**：首次在大规模VLA预训练中引入密集ECoT监督，并巧妙设计注意掩码使推理时跳过ECoT生成，兼顾表示质量和效率。
- **架构设计**：双流解耦（VLM负责推理，DiT动作专家负责连续控制），灵活且可扩展。
- **数据**：构建了大规模、高注释覆盖率的ECoT数据集（ProcCorpus-60M），自动化标注管道可复现。
- **跨本体通用性**：单一预训练权重可适应单臂、双臂、人形和真实机器人，表明表示对齐有效。
- **实验结果充分**：覆盖多个基准，对比方法全面，消融实验合理。

## 8. 不足与局限
- **预训练数据规模相对较小**：仅约1000小时，远低于π0（>10000小时）、LingBot-VLA（~20000小时），导致某些技能（如关柜门、关抽屉）表现差，数据分布有偏。
- **真实实验仅限一个平台（xArm）**，且任务数量有限（4个），缺乏更多真实机器人（如双机械臂、人形）的验证。
- **ECoT标注成本高**：需对每帧进行VLM前向推理，论文建议未来研究高效标注策略（如只选择关键帧）。
- **未进行更多消融实验**：例如不同ECoT组件的影响、不同注意掩码设计、不同VLM初始化等。
- **未见扩展至复杂任务或包含移动基座的机器人**：评估环境均为固定底座/表面对象操作，未涉及导航或移动操作。
- **潜在偏差**：ECoT自动标注可能引入标注器VLM的偏差；预训练数据主要来自开源数据集，可能覆盖特定场景和物体分布，泛化到全新场景需验证。
