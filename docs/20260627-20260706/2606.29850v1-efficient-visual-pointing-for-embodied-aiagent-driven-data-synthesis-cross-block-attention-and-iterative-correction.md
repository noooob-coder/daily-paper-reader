## 摘要
视觉指向将语言指令映射到像素坐标，是具身AI的核心技能。我们描述了我们的PointArena 2026解决方案，该方案实现了77.2%的整体准确率，并在基准测试中排名第二。该方法针对三种失败模式。首先，智能体驱动的合成构建了大规模语义和锚点相对候选池；服务器库存包含55,372个处理后的输出、53,772个去重样本ID和37,574个可训练完成或接受的行。其次，确定性可操控数据流水线使用掩码、模板和路径验证，创建了一个经过验证的10,000样本主集和备用样本。第三，两个模型侧模块解决了互补性错误：AttnRes通过门控跨块注意力增强可操控性，而ABC校正将扰动坐标与视觉特征相结合，用于通用坐标定位。类别感知路由将互补专家组合；用于选择专家的本地验证记录显示：Affordance为93.9%，Spatial Relation为82.6%，Reasoning为78.2%，Counting为70.4%，Steerability为63.0%。

## Abstract
Visual pointing maps a language instruction to pixel co ordinates, a core skill for embodied AI. We describe our PointArena 2026 solution, which achieves 77.2% overall accuracy and ranks second on the benchmark. The ap proach targets three failure modes. First, agent-driven syn thesis builds large semantic and anchor-relative candidate pools; the server inventory contains 55,372 processed out puts, 53,772 de-duplicated sample IDs, and 37,574 train able completed or accepted rows. Second, a determinis tic steerable-data pipeline creates a verified 10,000-sample main set, plus reserve samples, using masks, templates, and path verification. Third, two model-side modules address complementary errors: AttnRes adds gated cross-block at tention for steerability, while ABC correction encodes per turbed coordinates with visual features for general coordi nate grounding. Category-aware routing combines comple mentary specialists; local validation used to select experts records 93.9% Affordance, 82.6% Spatial Relation, 78.2% Reasoning, 70.4% Counting, and 63.0% Steerability.

### 1. 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：视觉指向（Visual Pointing）任务要求模型将自然语言指令精确映射到图像中的像素坐标，是具身AI（Embodied AI）的核心能力。现有模型存在三大瓶颈：
  - **数据稀缺**：公开的坐标标注数据以物体命名（Object Reference）为主，缺乏针对可操作性（Steerability）的直接监督。
  - **空间状态传播不足**：标准LoRA微调无法显式地在Transformer块之间传递锚点（anchor）状态，导致锚点相对推理能力弱。
  - **单次预测无法校正**：模型仅输出一个坐标样本，没有结构化方式来修正近误（near miss）。
- **整体含义**：本文旨在系统性地解决上述三个瓶颈，提升具身AI在复杂视觉指向任务中的准确率和鲁棒性。研究工作在PointArena 2026基准上排名第二，整体准确率达77.2%。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **关键技术细节**：
  - **智能体驱动数据合成（Agent-Driven Data Synthesis）**：
    - 两个数据流：语义流（使用Gemini、Qwen3-8B等LLM处理22.3M原始QA记录，经门控、分类、多点检测、重写四阶段）和可操作性流（使用确定性生成器，基于SAM风格跟踪器生成对象掩码、700个锚点相对模板，通过连通分量验证拒绝模糊路径）。

  - **AttnRes（跨块注意力）**：
    - 每四个Transformer块添加一个门控残差注意力模块。公式：\( H_i' = H_i + \tanh(\alpha) \cdot \text{AttnRes}(H_i; H_{i-3:i-1}) \)，其中门控标量α初始化为0，训练开始时保持预训练行为，仅在需要时激活历史路径。
    - 专门用于提升可操作性（Steerability），而非通用替换，因为消融表明其可能伤害非可操作性类别。
  - **ABC迭代校正**：
    - 训练模型从扰动点\(\tilde{p}=p^*+\epsilon\)恢复真实点\(p^*\)，输入包括图像、指令和扰动点的表示。
    - 三种表示：A（文本形式附加坐标）、B（用PointMLP编码扰动点并融合中间ViT特征）、C（添加密集多尺度坐标热图经小型CNN处理）。主要结论：视觉点编码（B和C）比额外校正轮次更重要，且最佳性能在校正第0轮达到。
  - **类别感知路由（Category-Aware Routing）**：
    - 验证分析显示没有单一检查点支配所有类别，因此最终提交使用路由：Affordance/Counting/Spatial → CoordMap专家，Reasoning → PointMLP+ViT专家，Steerability → AttnRes专家。
