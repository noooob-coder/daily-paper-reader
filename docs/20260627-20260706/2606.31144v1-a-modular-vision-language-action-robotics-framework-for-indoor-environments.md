## 摘要
本文提出了一个面向CMU视觉-语言-动作（VLA）挑战赛的集成系统，旨在使自主代理能够基于自然语言指令执行复杂任务。我们的框架采用模块化架构，协调环境映射、问题处理和导航。系统以两个并行流运行：感知流水线通过OwlViT嵌入从实时摄像头输入构建语义体素地图，语言流水线则使用视觉-语言模型对用户指令进行分类。映射受时间限制：若达到500秒的探索限制，系统将基于部分地图继续运行。随后，将分类后的查询置于地图的几何与语义上下文中，以生成详细的VLM提示。这产生了一个可执行的输出，展示了弥合人类语言与机器人动作之间差距的有效解决方案。

## Abstract
This paper presents an integrated system for the CMU Vision-Language-Action (VLA) Challenge, designed to enable an autonomous agent to perform complex tasks based on natural language instructions. Our framework employs a modular architecture that orchestrates environment mapping, question processing, and navigation. The system operates in two parallel streams: a perception pipeline that constructs a semantic voxel map from real-time camera feeds using OwlViT embeddings, and a language pipeline that classifies user commands with a Vision-Language Model. The mapping is time-constrained; the system proceeds with a partial map if a 500-second exploration limit is reached. The classified query is then grounded in the geometric and semantic context of the map to generate a detailed prompt for the VLM. This yields an actionable output, demonstrating a capable solution for bridging the gap between human language and robotic action.

## 2. 方法论

- **核心思想**：采用“先探索建图、再分类查询、最后生成动作”的序贯模块化架构。系统并行运行两条流水线：感知流水线（构建语义体素地图）和语言流水线（分类与推理），最终将查询与地图上下文融合，驱动VLM输出具体行动。
- **关键技术细节**：
  - **自主探索与2D栅格地图**：基于LiDAR构建2D占用栅格图；规划器通过采样可遍历点、计算最优路径（覆盖所有点），生成一系列航点；由航点控制器依次执行，实现全覆盖探索。
  - **语义体素地图构建**：在探索同时，使用OwlViT（google/owlvit-base-patch32）从实时摄像头图像中检测物体并生成特征嵌入；将嵌入、3D坐标和尺度逐步添加到体素地图中（体素大小可调）。探索有500秒硬限制，超时则使用部分地图。
  - **VLM驱动的查询处理**：
    1. **查询分类**：使用Gemini 2.0 Flash将自然语言指令分为三类：数值型、物体引用型、指令跟随型。
    2. **物体定位**：从指令中提取关键物体名词，计算文本嵌入，与体素地图中所有嵌入进行相似度匹配，找到对应3D点。
    3. **上下文提示生成**：将匹配到的物体名称、坐标、尺度及其在2D栅格图中的位置格式化为详细提示，供VLM推理。
  - **动作生成**：依据查询类型，解析VLM输出：
    - 数值型：返回整数答案。
    - 物体引用型：解析坐标并在RViz中发布Marker高亮物体。
    - 指令跟随型：解析VLM生成的高层计划（如“Goto(x,y), Avoidbetween(...)”），动态修改2D栅格地图，调用路径规划算法生成全局路径并执行。
- **算法/流程（文字描述）**：
  ```
  1. 启动后，LiDAR、相机持续采集数据。
  2. 探索规划器生成全覆盖航点序列 → 航点控制器驱动机器人。
  3. 探索过程中，物体检测节点处理相机帧 → OwlViT生成嵌入 → 语义映射节点逐步构建体素地图。

  5. 用户下达指令 → VLM分类 → 物体接地（检索体素地图） → 生成上下文提示 → VLM推理 → 解析输出 → 执行对应动作（导航/标注/返回答案）。
  ```

## 6. 主要结论与发现

- 模块化架构（感知+语言+行动）结合时间受限的主动建图，能够在小规模室内场景中完成任务。
